# Communication Channels

Last updated: 2026-06-12

## Decision

iGentic uses a **GitHub-first communication model**.

The project may publish on Instagram, X and LinkedIn, but these channels point back to GitHub. They do not replace issues, pull requests, discussions, security reports or roadmap files.

## Official channel map

| Channel | Use now | Use later | Notes |
| --- | --- | --- | --- |
| GitHub README | Yes | Yes | Main public entry point |
| GitHub Issues | Yes | Yes | Bugs, feature proposals, design feedback, research tasks |
| GitHub Pull Requests | Yes | Yes | Code, docs, tests, assets |
| GitHub Discussions | Optional | Yes | Enable when there is repeated community discussion pressure |
| Instagram | Yes | Yes | Visual build-in-public and design story |
| X | Yes | Yes | Short research updates and developer discovery |
| LinkedIn | Yes | Yes | Professional positioning and serious privacy/AI posts |
| Discord | No | Later | Only after GitHub activity needs real-time coordination |
| YouTube / Shorts | No | Later | Architecture explainers and device test demos |
| Newsletter | No | Later | Monthly digest after there is consistent progress |

## Channel principles

1. Link back to GitHub for anything actionable.
2. Keep public claims accurate and evidence-based.
3. Do not post private data, secrets, customer-style anecdotes or real user messages.
4. Avoid Apple brand confusion.
5. Make the experimental status clear.
6. Prefer source links over vague claims.

## Instagram role

Instagram is for:

- logo/design evolution,
- simple architecture visuals,
- build-in-public carousels,
- community calls for designers and contributors,
- explaining local-first AI to non-experts.

Instagram should not be used for:

- accepting feature decisions,
- security reports,
- private debugging,
- screenshots with real private data.

## X role

X is for:

- short technical updates,
- links to issues and PRs,
- research notes,
- release announcements,
- finding local-AI and Swift contributors.

X should not be the canonical project log.

## LinkedIn role

LinkedIn is for:

- serious positioning,
- privacy-first AI posts,
- open-source collaboration invitations,
- longer status updates,
- explaining why local control matters.

## Discord decision

Do not launch Discord yet.

Reason: an early Discord can fragment decisions before the repo has enough contributors and maintainers. Enable Discord only when:

- GitHub Issues and Discussions have repeated active participants,
- there are at least two non-maintainer contributors,
- there is a clear moderation plan,
- decisions remain mirrored back to GitHub.

## Security communication

Security issues should follow `SECURITY.md`.

Do not report secrets, vulnerabilities or private user data on Instagram, X, LinkedIn or public GitHub issues.

## Social account naming

Preferred handles if available:

- `@igenticai`
- `@igenticdev`
- `@igentic`
- `@igentic_project`

Avoid handles that start with Apple trademarks or imply official Apple affiliation.

## Canonical link policy

Every social bio should link to the GitHub repository or a future official landing page that links to GitHub.

Suggested bio:

> Open-source personal AI control layer. Local-first. Approval-gated. Audit-ready. Starting on iPhone.

## Review cadence

Review this channel policy whenever:

- a new official channel is created,
- Discord or Discussions are enabled,
- the project publishes a release,
- the brand direction changes,
- community moderation load increases.
