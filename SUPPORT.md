# Support

Thank you for using or exploring iGentic iPhone.

This repository is an experimental open-source research prototype for a privacy-first iPhone AI Runtime Layer. It is not a production product and does not provide professional advice.

## Where to ask questions

Use GitHub issues for:

- architecture questions,
- documentation improvements,
- reproducible bugs,
- iPhone test reports,
- model/runtime evaluation notes,
- proposal discussions.

Use the issue templates when possible:

- Feature request
- Security review
- Model candidate

## Before opening an issue

Please check:

- `README.md`,
- `AGENTS.md`,
- `PROJECT_STATE.md`,
- `docs/CHATGPT_NEXT_TASK.md`,
- `docs/SPARSAMKEIT.md`,
- existing open issues.

## Good issue reports

A useful issue includes:

- expected behavior,
- actual behavior,
- reproduction steps,
- device or simulator details,
- iOS version when relevant,
- whether real device testing was done,
- logs only if they contain no private data.

## Privacy warning

Do not paste private data into public issues.

Do not share:

- contacts,
- messages,
- calendar entries,
- health data,
- financial data,
- credentials,
- API keys,
- provisioning profiles,
- private documents,
- raw personal logs.

Use synthetic data in examples.

## Security reports

For security or privacy-sensitive problems, follow `SECURITY.md`.

## Scope of support

We can discuss:

- repository structure,
- Swift package behavior,
- policy and approval design,
- safe local-first architecture,
- documentation,
- reproducible tests.

We do not provide:

- medical advice,
- legal advice,
- financial advice,
- support for handling real private datasets in public,
- Apple Developer account or signing credential handling in public issues.
